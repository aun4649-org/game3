# エディタ仕様 — 対話型インタプリタ実行環境（COSMOS II）

本仕様は「対話型インタプリタ実行環境（COSMOS II）」における、プログラムコードエディタ（ダイレクトモードおよびフルスクリーンエディタ機能）の仕様を定義する。

---

## 1. 画面構成および基本機能

### 1.1 ターミナル画面
- **画面サイズ**: 80桁 × 25行
- **表示形態**: フルスクリーンレトロターミナル風（CRTエフェクト付き）
- **カーソル**: 現在の入力位置を示す点滅カーソルを表示。

### 1.2 キーボード操作と文字入力
- **文字入力**: 印字可能文字はそのままカーソル位置に挿入される（大文字・小文字を区別する）。
- **カーソル移動**: 上下左右の矢印キーで画面内のカーソルを移動可能。プロンプト（`>`）よりも左には移動できない。
- **削除操作**:
  - `Backspace`: カーソル位置の直前の文字を消去し、以降の文字列を左にシフトする。
  - `Delete`: カーソルの現在位置の文字を消去し、以降の文字列を左にシフトする。
- **改行・コマンド実行**: `Enter` キー押下で、カーソル行が評価・実行され、カーソルが次の行へ移動する（最下行の場合は画面がスクロールする）。

### 1.3 ページャー機能
- プログラムリスト出力など、1画面（25行）に収まらない出力をする場合、下部に `-- MORE --` を表示し一時停止する。
- 任意のキー入力で次のページを表示する。

---

## 2. コマンド処理（ダイレクトモード）

`Enter` キー押下時、入力された行の内容に応じて以下のいずれかの処理が行われる。

### 2.1 行番号付き入力（プログラムの入力と削除）
行頭が数字（1〜32767）で始まる場合、プログラムコードの入力とみなす。

- **行の追加・更新**: `[行番号] <文>` （例：`10 A=1`）
  - 指定された行番号のプログラムとしてメモリ（`programMemory`）に保存される。
  - プログラム行の先頭にスペースが含まれる場合は、最初のスペース1つがパース時に考慮され、それ以降がコード本体として格納される（例: `10  A=1` -> ` A=1`）。
- **行の削除**: `[行番号]` （文字列部分が空の場合）
  - 指定された行番号のプログラムをメモリから削除する。

> [!WARNING]
> 行番号が 1〜32767 の範囲外の場合、`LINE NUMBER OUT OF RANGE (1-32767)` というエラーを出力し、保存しない。

### 2.2 直接実行文（行番号なし入力）
行頭が数字で始まらず、かつシステムコマンドに該当しない場合、入力された行は「行番号なしの文（ダイレクトステートメント）」として扱われる。

- **評価・実行**: 入力行は即座に COSMOS 言語のパーサおよびエバリュエータに渡され、実行される。
  - 例1: `? 1 + 2 * 3` と入力すると、直ちに `7` が出力される。
  - 例2: `A = 10` と入力すると、変数 `A` に `10` が記憶される。
- これにより、電卓的な利用や変数状態の確認・操作を対話的に行うことができる。

### 2.3 システムコマンド

システムを管理する組み込みコマンド。すべて大文字でも小文字でも認識されるが、内部的に大文字に変換されて処理される。

| コマンド | 引数仕様 | 機能説明 |
|----------|----------|----------|
| `LIST` | `[開始行]-[終了行]` (省略可) | メモリ内のプログラムリストを行番号順に表示。範囲指定可能（例: `LIST 10-50`、`LIST 100-`、`LIST -20`）。プログラムが無い場合は `NO PROGRAM` と表示。 |
| `DELETE` | `[開始行]-[終了行]` (必須) | 指定範囲のプログラム行を削除する。完了時は `OK`、該当行が無い場合は `LINE NOT FOUND`、引数が無い場合は `SYNTAX ERROR` を表示。 |
| `NEW` | なし | プログラムメモリおよび全変数を初期化（消去）する。完了時 `OK` と表示。 |
| `#=` | `[行番号]` | 入力された式の値を評価し、指定された行番号からプログラムの実行を開始（RUN）する。（言語仕様の GOTO `#=<式>` と同等の振る舞いで起動）。 |
| `CLEAR` または `CLS`| なし | ターミナルの画面をクリアする。 |
| `COLOR` | `W`\|`G`\|`A` （White/Green/Amber） | ターミナルの文字色（テーマ）を変更する。完了時 `OK` と表示。引数エラー時は `USAGE: COLOR WHITE|GREEN|AMBER` を表示。 |
| `FONT` | `<整数サイズ>` | ターミナルのフォントサイズを変更する（5px〜99px）。完了時 `OK` と表示。 |
| `HELP` | なし | コマンドの一覧と簡単な説明を表示する（ここでは `#=[行番号]` による実行について案内する）。 |

## 3. インタープリタ連携（Runtime 呼び出し）

- **実行起動時の動作**: ダイレクトモードの文として `#=<式>` が入力・評価された際、評価値（行番号）を開始地点としてインタープリタ（`cosmosRun`）へ内部メモリモデルの状態（`programMemory` および 64KB `memory`）を引き渡す。
- **実行中のI/O処理と制御**:
  - `runtimePrint` / `runtimePrintln` による文字出力はすべてターミナルバッファを通じて画面に描画される。
  - `runtimeInputChar` (`$`) / `runtimeInputNumber` (`?`) による入力待機中は、エディタの通常のカーソル移動や編集機能はロックされ、インタープリタへの入力（数値のエンター確定または単一文字の即時確定）のみが受け付けられる。
  - **強制終了（ブレーク）**: プログラム実行中（入力待機中も含む）に `CTRL+C` が押下された場合、実行エンジンに対して中断シグナル（フラグなど）を立てる、もしくは即座に Promise を Reject して実行ループを抜け、エディタ・コマンド入力状態に復帰する。
