# 独自言語仕様 — 対話型インタプリタ実行環境（COSMOS II）【符号付整数版】

以下の独自言語仕様に基づいて「対話型インタプリタ実行環境（COSMOS II）」を構築する。
プログラムコード作成部分（エディタ）はすでに実装済みであるため、本仕様はプログラムコードの実行制御部分の追加実装を対象とする。

> [!IMPORTANT]
> 本仕様は**16bit 符号付整数版**である。数値範囲は **-32768 〜 32767** となる。

---

## 1. データモデル

### 1.1 数値型

- すべての値は **16bit 符号付整数（-32768 〜 32767）** として扱う。
- 内部表現は**2の補数**形式とする。
- 演算結果が範囲外となる場合は **16bit幅でラップアラウンド**（2の補数のオーバーフロー動作）する。
  - 例: `32767 + 1 = -32768` / `-32768 - 1 = 32767`

### 1.2 変数

| 種別 | 識別子 | 属性 | 初期値 |
|------|--------|------|--------|
| ユーザー変数 | `A`〜`Z`, `a`〜`z`（計52個、1文字のみ識別） | 読み書き | `0` |
| システム変数 `%` | 直近の除算 `/` における剰余 | 読み取り専用 | `0` |
| システム変数 `'` | 参照のたびに -32768〜32767 の乱数を生成 | 読み取り専用 | 不定（動的生成） |

> [!IMPORTANT]
> システム変数 `%`, `'` への代入はエラーとする（§5.1 参照）。

### 1.3 メモリモデル

インタープリタが管理する仮想メモリは **64KB**（`$0000`〜`$FFFF`）とし、以下のレイアウトで管理する。

| アドレス範囲 | 領域名 | 用途 |
|-------------|--------|------|
| `$0000`〜`$0067` | 変数領域 | ユーザー変数 A-Z, a-z を各2バイトで連続配置 |
| `$0068`〜`$00FF` | 予約領域 | 将来のシステム拡張用 |
| `$0100`〜`$7FFF` | フリー領域 | 配列参照で利用可能なユーザー自由領域 |
| `$8000`〜`$FFFF` | プログラム領域 | プログラムソースコード |

システム変数およびインタープリタ実行用スタックは、上記メモリマップとは独立した領域で管理する。

#### 変数アドレスの配置

```
A = $0000, B = $0002, ..., Z = $0032
a = $0034, b = $0036, ..., z = $0066
```

#### バイトオーダー

変数の値は**ビッグエンディアン**で保持する。

`A = $1234`（10進: 4660）のとき:

| 参照 | アドレス | 値 |
|------|---------|------|
| バイト参照 `A[0]` | `$0000` | `$12`（上位バイト） |
| バイト参照 `A[1]` | `$0001` | `$34`（下位バイト） |
| ワード参照 `A:0]` | `$0000`-`$0001` | `$1234`（符号付: 4660） |
| ワード参照 `A:1]` | `$0002`-`$0003` | 隣接変数Bの値 |

`A = -1`（内部表現: `$FFFF`）のとき:

| 参照 | アドレス | 値 |
|------|---------|------|
| バイト参照 `A[0]` | `$0000` | `$FF` |
| バイト参照 `A[1]` | `$0001` | `$FF` |
| ワード参照 `A:0]` | `$0000`-`$0001` | `$FFFF`（符号付: -1） |

> [!WARNING]
> 実行環境（x86/x64等）はリトルエンディアンが多いため、`Memory[]` 配列への読み書き時に明示的なバイトスワップが必要。

> [!NOTE]
> バイト参照 `A[n]` で読み出す値は常に 0〜255（無符号8bit）である。ワード参照 `A:n]` で読み出す値は符号付16bit（-32768〜32767）として解釈する。

## 2. 字句定義（Lexical）

### 2.1 トークン種別

| トークン | 表記 | 規則 |
|---------|------|------|
| 10進定数 | `0`〜`32767` | 数字のみで構成。負の定数はリテラルとしては存在せず、単項 `-` で表現する（例: `-1`）。 |
| 16進定数 | `$` + 16進数字 | 1〜4桁、`0`-`9`, `A`-`F`（大文字・小文字混在可）。値は2の補数として解釈する（例: `$FFFF` = -1）。 |
| 2進定数 | `%` + 2進数字 | 1〜16桁、`0` または `1` のみ。値は2の補数として解釈する。 |
| 文字定数 | `"` + ASCII文字 + `"` | 評価値はその文字のASCIIコード（0〜127）。 |
| 変数名 | `A`〜`Z`, `a`〜`z` | 1文字のみ識別（2文字目以降は無視） |
| システム変数 | `%`, `'` | §1.2 参照 |
| 演算子 | `+`, `-`, `*`, `/`, `=`, `<>`, `<`, `>`, `<=`, `>=`, `&&`, `\|\|`, `^`, `<<`, `>>`, `#`, `##` | §4 参照 |
| 区切り記号 | `(`, `)`, `[`, `]`, `:`, `,`, `;`, `@`, `!`, `?`, `$`, `.`, `/`, `"` | 文脈により意味が変わる |

### 2.2 `%` の曖昧性解決

`%` は2進定数のプレフィクスとシステム変数の2つの意味を持つ。以下の規則で判別する：

- `%` の直後が `0` または `1` → **2進定数**として解釈
- `%` の直後が `0`, `1` 以外（空白、演算子、行末等）→ **システム変数**として解釈

## 3. プログラム構造（Program Structure）

### 3.1 行の構造

```bnf
<行>       ::= <行番号> " " <行本体> <改行>
<行番号>   ::= "1" 〜 "32767"
<改行>     ::= "\n" | "\r\n"
<行本体>   ::= <文の並び> | <コメント>
<文の並び> ::= <文> { <空白> <文> }
<空白>     ::= { " " }+
```

### 3.2 コメントの判定

行本体の先頭文字が、いかなる文の開始記号にも該当しない場合、その行全体をコメントとして扱う。

**文の開始記号一覧**: `A`-`Z`, `a`-`z`, `#`, `!`, `@`, `;`, `?`, `$`, `.`, `/`, `"`, `]`

## 4. 構文定義と動作仕様

### 4.1 式と演算子

#### 構文

```bnf
<式>   ::= <項> { <二項演算子> <項> }
<項>   ::= <定数>
         | <変数名>
         | <配列参照>
         | <システム変数>
         | <単項演算子> <項>
         | "(" <式> ")"
         | "?"                (* 10進数値入力: ユーザー入力を数値として返す *)
         | "$"                (* 1文字入力: 入力文字のASCIIコードを返す *)

<定数> ::= <10進定数> | <16進定数> | <2進定数> | <文字定数>
```

#### 評価順序

1. **最優先**: 括弧 `(式)`、定数、変数、配列参照、システム変数、入力 `?` / `$`
2. **次点**: 単項演算子（右から左に結合。例: `-#1` は `-(#1)`）
3. **最後**: 二項演算子（**左から右に順次評価。種類による優先順位の差はない**）

例:
- `1 + 2 * 3` → `(1 + 2) * 3` → `9`
- `0 && 1 || 1` → `(0 && 1) || 1` → `1`

#### 二項演算子

```bnf
<二項演算子> ::= <算術演算子> | <比較演算子> | <ビット演算子> | <シフト演算子>
```

**算術演算子**

| 構文 | 動作 |
|------|------|
| `+` | 加算。結果は16bit符号付でラップアラウンド。 |
| `-` | 減算。結果は16bit符号付でラップアラウンド。 |
| `*` | 乗算。結果は16bit符号付でラップアラウンド。 |
| `/` | **符号付整数除算**（ゼロへの丸め）。実行直後、システム変数 `%` に剰余を格納する。剰余の符号は被除数と同じ。右辺が `0` の場合は実行時エラー（§5.2）。例: `7 / 2 = 3`（`% = 1`）、`-7 / 2 = -3`（`% = -1`）。 |

**比較演算子**

| 構文 | 動作 |
|------|------|
| `=` | 等しければ `1`、そうでなければ `0` |
| `<>` | 等しくなければ `1`、そうでなければ `0` |
| `<` | 左辺 < 右辺 なら `1`、そうでなければ `0`（**符号付比較**） |
| `>` | 左辺 > 右辺 なら `1`、そうでなければ `0`（**符号付比較**） |
| `<=` | 左辺 <= 右辺 なら `1`、そうでなければ `0`（**符号付比較**） |
| `>=` | 左辺 >= 右辺 なら `1`、そうでなければ `0`（**符号付比較**） |

> [!NOTE]
> 比較は符号付として行う。例: `-1 < 1` は `1`（真）。無符号版では `$FFFF > 1` となり結果が異なる。

**ビット演算子**

| 構文 | 動作 |
|------|------|
| `&&` | ビットワイズ AND（ビットパターンに対して演算。結果は符号付として解釈） |
| `\|\|` | ビットワイズ OR |
| `^` | ビットワイズ XOR |

**シフト演算子**

| 構文 | 動作 |
|------|------|
| `<<` | 左シフト。結果は16bit符号付でラップアラウンド。 |
| `>>` | **算術右シフト**。符号ビットを維持しながら右シフトする。例: `-4 >> 1 = -2`。 |

#### 単項演算子

| 構文 | 動作 |
|------|------|
| `+` | 絶対値。負の値は正に変換する。例: `+(-5) = 5`。ただし `+(-32768) = -32768`（表現不可のためラップアラウンド）。 |
| `-` | 符号反転。`0 - x` として計算（2の補数）。例: `-(5) = -5`。ただし `-(-32768) = -32768`（ラップアラウンド）。 |
| `#` | 論理否定。値が `0` なら `1`、それ以外なら `0`。 |
| `##` | ビット反転。全ビットを反転する。例: `##0 = -1`、`##(-1) = 0`。 |

---

### 4.2 配列参照

```bnf
<配列参照>   ::= <バイト参照> | <ワード参照>
<バイト参照> ::= <変数名> "[" <式> "]"
<ワード参照> ::= <変数名> ":" <式> "]"
```

| 種別 | アドレス計算 | アクセス単位 |
|------|-------------|-------------|
| バイト参照 `V[n]` | `V` の値 + `n` | 1バイト（読み出し: 0〜255、書き込み: 下位8bit） |
| ワード参照 `V:n]` | `V` の値 + `n` × 2 | 2バイト（符号付16bit、ビッグエンディアン） |

アドレス計算は無符号として行う（変数値のビットパターンを無符号アドレスとして使用）。算出アドレスが `0`〜`65535` の範囲外の場合は実行時エラー（§5.2）。

### 4.3 代入文

```bnf
<代入文>   ::= <代入対象> "=" <式>
<代入対象> ::= <変数名> | <配列参照>
```

---

### 4.4 制御文

```bnf
<制御文> ::= "#" "=" <式>      (* GOTO *)
           | "!" "=" <式>      (* GOSUB *)
           | "]"               (* RETURN *)
           | ";" "=" <式>      (* IF *)
```

| 文 | 動作 |
|----|------|
| `# = <式>` (GOTO) | 式の値が `-1`（内部表現 `$FFFF`）ならプログラム終了（END）。それ以外は、式の値を行番号として解釈し、指定行番号へジャンプする。指定行番号が存在しない場合、それより大きい最小の行番号へジャンプする。該当する行が存在しない場合は実行時エラー（§5.2）。 |
| `! = <式>` (GOSUB) | 現在の実行位置（行番号＋文インデックス）をCallStackに保存し、GOTO と同じ規則で指定行へジャンプする。 |
| `]` (RETURN) | CallStack から戻り先を取り出し、呼び出し元の次の文から実行を再開する。スタックが空の場合は実行時エラー（§5.2）。 |
| `; = <式>` (IF) | 式の結果が `0`（偽）の場合、現在の行の残りの文をすべてスキップし、次の行へ移行する。`0` 以外（真）の場合は何もせず次の文を実行する。 |

> [!NOTE]
> 行番号は `1`〜`32767` であるため、GOTO/GOSUB の式の値が負数（-1 以外）や 0 の場合、有効な行番号には該当しない。`-1` は特別にプログラム終了として扱う。

---

### 4.5 ループ文

```bnf
<ループ文> ::= "@"                         (* DO: ループ開始点 *)
             | "@" "=" "(" <式> ")"        (* UNTIL: 条件真で終了 *)
             | <代入対象> "=" <式> "," <式> (* FOR: 初期値, ステップ *)
             | "@" "=" <式>                (* NEXT: 上限値 *)
```

#### DO 〜 UNTIL

1. `@` — 現在の実行位置（行番号＋文インデックス）を LoopStack に保存する。
2. `@=(<式>)` — 式を評価し、結果が `0`（偽）なら LoopStack の先頭が指す `@` の次の文に戻る。`0` 以外（真）なら LoopStack から先頭を取り除き、次の文へ進む（ループ終了）。

#### FOR 〜 NEXT

1. `<変数> = <初期値>, <ステップ>` — 変数に初期値を代入し、ステップ値を内部に記憶する。同時に、現在の実行位置を LoopStack に保存する。
2. `@ = <上限値>` — 変数にステップ値を加算する。加算後の変数値が上限値を超えていなければ、FOR 設定行の次の文に戻る。超えていればループ終了（LoopStack から先頭を取り除き、次の文へ進む）。

> [!NOTE]
> 「超えている」の判定はステップ値の符号で決まる。ステップが正なら `変数 > 上限値` で終了、ステップが負なら `変数 < 上限値` で終了。

> [!NOTE]
> パーサは代入文のパース時に `,` が後続すれば FOR 文として判別する。

---

### 4.6 入出力文

```bnf
<入出力文> ::= "?" "=" <式>                    (* 10進出力 *)
             | "??" "=" <式>                   (* 16進4桁出力 *)
             | "?$" "=" <式>                   (* 16進2桁出力 *)
             | "?0" "(" <数字> ")" "=" <式>    (* n桁右詰め出力、ゼロパディング *)
             | "?" "(" <数字> ")" "=" <式>     (* n桁右詰め出力、空白パディング *)
             | "$" "=" <式>                    (* 1文字出力 *)
             | "." "=" <式>                    (* 空白出力 *)
```

| 文 | 動作 |
|----|------|
| `? = <式>` | 式の値を**符号付10進数**で出力する。負の値は `-` 付きで表示される。例: `? = -5` → `-5` |
| `?? = <式>` | 式の値の内部表現を16進4桁（ゼロパディング）で出力する。例: `?? = -1` → `FFFF` |
| `?$ = <式>` | 式の値の下位8bitを16進2桁（ゼロパディング）で出力する。例: `?$ = -1` → `FF` |
| `?0(n) = <式>` | 式の値を符号付10進n桁で右詰め出力する（ゼロパディング）。符号は桁数に含む。 |
| `?(n) = <式>` | 式の値を符号付10進n桁で右詰め出力する（空白パディング）。符号は桁数に含む。 |
| `$ = <式>` | 式の値の下位8bitをASCII文字として1文字出力する。 |
| `. = <式>` | 式の値が正の場合、その数だけ半角スペースを出力する。`0` 以下の場合は何も出力しない。 |

### 4.7 文字列出力・改行出力

```bnf
<文字列出力> ::= "\"" <文字列> "\""    (* ダブルクォート内の文字列をそのまま出力 *)
<改行出力>   ::= "/"                  (* 改行を出力 *)
```

### 4.8 文の定義

```bnf
<文> ::= <代入文>
       | <制御文>
       | <ループ文>
       | <入出力文>
       | <文字列出力>
       | <改行出力>
```

## 5. エラー仕様

### 5.1 構文エラー（SYNTAX ERROR）

字句解析・構文解析フェーズで不正な記述を検出した場合、実行を開始せずにエラーを出力する。

| 要因 | エラーメッセージ |
|------|------------------|
| 未定義の記号 | `** SYNTAX ERROR : Undefined Symbol` |
| 閉じられていない括弧 `( )` | `** SYNTAX ERROR : Unclosed bracket` |
| システム変数（`%`, `'`）への代入試行 | `** SYNTAX ERROR : Attempt to assign to a system variable` |
| 行頭以外での行番号記述 | `** SYNTAX ERROR : Line no. are only allowed at line start` |
| その他の構文違反 | `** SYNTAX ERROR : Undefined syntax` |

### 5.2 実行時エラー（RUNTIME ERROR）

プログラム実行中に以下の状態が発生した場合、直ちに実行を停止し、エラーメッセージを表示してエディタに戻る。

| 要因 | エラーメッセージ |
|------|------------------|
| `/` の右辺が `0` | `** RUNTIME ERROR : Division by zero` |
| 配列参照のアドレスが 0〜65535 範囲外 | `** RUNTIME ERROR : Address out of range` |
| CallStack が空の状態で `]` を実行 | `** RUNTIME ERROR : Stack underflow` |
| `!`（GOSUB）のネストが実装上限を超過 | `** RUNTIME ERROR : Stack overflow` |
| 対応する `@` なしで `@=(式)` / `@=値` を実行 | `** RUNTIME ERROR : Mismatched control blocks` |
| GOTO/GOSUB 先の行が存在せず、より大きい行番号もない | `** RUNTIME ERROR : Mismatched control blocks` |

### 5.3 入力エラー

- **数値入力 `?`**: 数値以外の入力、または 16bit 符号付整数の範囲外（-32768〜32767 以外）の値が入力された場合は、エラーとせず、再入力を促すプロンプトを表示して待機する。

### 5.4 エラー表示形式

すべてのエラーは以下の3行形式で表示する：

```
<エラーメッセージ>
<エラーが発生した行の内容>
<エラー箇所の列位置に "^" を表示>
```

例:

```
** SYNTAX ERROR : Undefined Symbol
10 A = B + @
           ^
```

## 6. 実装ガイド

### 6.1 モジュール構成

| モジュール | 役割 |
|-----------|------|
| **Tokenizer** | ソースコードをトークンに分解する。`%` の曖昧性解決（§2.2）もここで行う。 |
| **Parser** | トークン列を解析し、文の種類（代入、制御、ループ、入出力等）を判別する。構文エラーを検出する。 |
| **Evaluator** | 式を左から右へ順次評価する。括弧の再帰的処理と単項演算子の即時結合を処理する。すべての演算は符号付16bitで行う。 |
| **Executor** | PC（行番号）と StatementIndex（文番号）の2階層ポインタで実行を制御する。ジャンプ、サブルーチン、ループの遷移を管理する。 |

### 6.2 実行ループのアルゴリズム

1. **行の特定**: PC が指す行番号を取得する。
2. **文の分割**: 行内の文をトークンリストとして取得する。
3. **逐次実行**:
   - StatementIndex が指す文を1つ実行する。
   - IF文（`;=`）が偽なら、行の残りの文を破棄してステップ1へ（PCを次の行に進める）。
   - GOTO/GOSUB なら PC を書き換え、ステップ1へ。
   - RETURN なら CallStack から復帰先を取得し、その位置から再開。
   - ループ制御（`@`, `@=`）なら LoopStack を操作し、条件に応じてジャンプ。
   - 上記以外なら StatementIndex をインクリメントし、次の文へ。
4. **終了判定**: PC が `-1`(END) に達するか、プログラム末尾を超えたら終了。

### 6.3 実行コンテキスト（Context）

実行時に保持すべきデータ構造:

| データ | 型 | 説明 |
|--------|-----|------|
| `Memory` | `uint8[65536]` | 64KBのバイト配列。変数・フリー領域・プログラム領域を含む。アドレスは常に無符号。 |
| `PC` | `int16` | 現在実行中の行番号。`-1` はプログラム終了を示す。 |
| `StatementIndex` | `int16` | 行内での文番号（0始まり）。 |
| `CallStack` | `Stack<(PC, StatementIndex)>` | GOSUB の戻り先を保持する LIFO スタック。 |
| `LoopStack` | `Stack<(PC, StatementIndex)>` | DO(`@`) / FOR の開始位置を保持するスタック。 |
| `SystemVar_Remainder` | `int16` | システム変数 `%` の値（直前の除算の剰余）。 |

### 6.4 無符号版との主な差異

| 項目 | 無符号版 | 符号付版（本仕様） |
|------|---------|-------------------|
| 数値範囲 | 0〜65535 | -32768〜32767 |
| 比較演算 | 無符号比較 | 符号付比較 |
| 右シフト `>>` | 論理シフト（0埋め） | 算術シフト（符号ビット維持） |
| 除算 `/` | 無符号除算 | 符号付除算（ゼロへの丸め） |
| 単項 `+` | 恒等演算 | 絶対値 |
| 10進出力 `?` | 0〜65535 を出力 | 負号付きで -32768〜32767 を出力 |
| 空白出力 `.` | 値の数だけ出力 | 正の値のみ出力（0以下は無出力） |
| END条件 | `# = 65535` | `# = -1` |
| 乱数 `'` | 0〜65535 | -32768〜32767 |
| 10進定数 | 0〜65535 | 0〜32767（負数は `-` 演算子で表現） |
